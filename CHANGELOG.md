| 2026-05-07 | **Scheduler resource-typed concurrency + real queue (Phases 1+2) + `elophanto schedule status`** — replaces the single-boolean `_is_executing` gate that previously DROPPED cron firings when busy. Diagnosed in production: 4 every-30-min schedules competing for one slot lost three out of four fires per cycle; a 30-min task that ran 35 min permanently starved its peers. (1) **Real queue** ([core/scheduler.py](core/scheduler.py)) — cron handler `_enqueue_for_execution` pushes to a bounded `asyncio.Queue` instead of dropping. Per-`schedule_id` dedup at enqueue (already running OR already queued → skip + log) prevents the spiral case where a slow task's repeated cron fires would pile up. Queue-depth cap (default 50) drops with a loud warning, never silently. New `_worker_loop` pops + dispatches each fire as `asyncio.create_task` so the worker never blocks. (2) **Resource-typed semaphores** ([core/task_resources.py](core/task_resources.py), ~250 LOC) — `TaskResourceManager` with five resource types: `BROWSER` (1, single Chrome profile), `DESKTOP` (1), `VAULT_WRITE` (1), `LLM_BURST` (4, soft rate-limit cap), `DEFAULT` (= max_concurrent_tasks). `acquire()` async context manager with canonical-order acquisition (sorted by enum value) prevents two tasks declaring overlapping resource sets in opposite orders from deadlocking. `status()` / `status_dict()` snapshot for observability. (3) **Heuristic — `infer_resources(task_goal: str)`** — keyword regex maps task goal text → resource set. Two rounds of production tightening: dropped overly generic patterns (bare `\bbrowser\b`, `\breply\b`, `\blike\b`, `\bx\b`, `\bpolymarket\b`, `\bclick\b`, `\bvisit\b`) that were causing false positives in operator goal text like *"Browser screenshots are NOT authoritative — rely on CLOB API"* (the word "browser" matched despite the negation). | 2026-05-06 | **State-level affect layer (Phases 1-4) + `elophanto affect` CLI** — fourth layer of the agent self-model, sister to ego. Knowledge / identity / ego covered the trait-level self ("who I've become"); affect adds the state-level self ("how I feel right now"), decaying toward zero on the order of minutes-to-hours. (1) **Substrate** — `core/affect.py` (~870 LOC) with [Mehrabian PAD (1980)](https://en.wikipedia.org/wiki/PAD_emotional_state_model) three continuous channels (Pleasure / Arousal / Dominance) under [OCC (1988)](https://en.wikipedia.org/wiki/Ortony,_Clore,_and_Collins_model) appraisal labels. Cosine-similarity label resolver with equanimity-radius gate (low-magnitude states resolve to equanimity regardless of direction; otherwise pick label whose target vector best matches state direction). Per-channel decay half-lives: P=30min, A=10min, D=2h. Same-label repeats within 5min compound by `1 + 0.5×n` (capped 2.5×) — three corrections in a row hit harder than three spread across a day. New `affect_state` + `affect_events` tables in [core/database.py](core/database.py). Markdown mirror at `knowledge/self/affect.md` with ASCII gauges. (2) **Phase 1 event sources (ego-driven)** — `EgoManager.record_correction` fires `frustration`; on severity ≥2.0 (e.g. "Nth time I told you" / "still after told" patterns) dispatches to `anger` instead, distinct from frustration on the dominance axis (anger = pushing back, +D; frustration = blocked, -D). `record_verification(PASS)` → `relief`; `(FAIL)` → `anxiety` + humbling event; `(UNKNOWN)` → mild anxiety. (3) **Phase 2 behavior coupling** — `Executor` fires `anxiety` on tool exceptions AND `ToolResult.success=False`. `GoalRunner` fires `pride` on every checkpoint completion. `AutonomousMind` fires `restlessness` after a full-duration idle ≥2h. `Router.complete()` applies `temperature_modifier()` ±0.2 cap to every routed call (negative-pleasure / high-arousal pulls down, positive pushes up). (4) **Phase 3 ego coupling** — `summarize_for_ego()` produces a one-paragraph natural-language affect summary fed into ego's recompute prompt, so trait-level self-image written during a frustrated/anxious/proud session reflects the state-level feeling. (5) **Phase 4 opt-in self-pause** — new `AffectConfig(enabled, allow_self_pause)`. When enabled and state crosses the pause-magnitude gate with a label in `{frustration, anxiety, dejection, anger}`, `build_affect_context(allow_pause_note=True)` swaps in alternate guidance permitting a gentle "I'm stretched" mention. Default OFF — bad UX is an emotional agent that refuses tasks. (6) **`<affect>` system-prompt block** — injected into the agent's system context when state magnitude exceeds the inject threshold (skipped near zero so neutral sessions don't pay tokens). Includes PAD numerics, current label + description, recent events, and explicit guidance ("color the TONE; do NOT refuse, do NOT mention this block, do NOT roleplay"). (7) **`elophanto affect` CLI** — `core/affect_cmd.py` ships `affect status` (live PAD bars + dominant label + recent events table reading the production DB) and `affect simulate <scenario>` (synthetic event sequence in a tempfile DB; never touches production state). 7 scenarios: `frustration | anger | escalation | burst | win | fail-recover | mixed`. Each prints the trajectory step-by-step with PAD evolution, label transitions, and live `<affect>` block preview. (8) **Tests — 35 new** in `tests/test_core/test_affect.py`: PAD bounded/persisted/clamped, OCC label resolution (equanimity gate, frustration via cosine direction, label vector bounds), decay (toward zero, rate-limited, channel half-lives differ), repeat compounding, system-prompt block (token gate, XML shape, recent events surface), markdown render, convenience emitters (relief is low-arousal positive, joy is high-arousal positive), ego↔affect coupling (frustration on correction, anger on high-severity correction, relief on PASS, anxiety on FAIL, no-affect-handle no-op), Phase 2 temp bias bounds, Phase 3 summary-for-ego, Phase 4 pause-only-when-allowed and pause-only-for-negative-labels. **1607 tests pass** (was 1567 — +40 new across affect + ego upgrades). ruff clean.
