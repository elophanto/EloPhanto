## Instructions

### Keyword Research and Optimization
1. Use `web_search` to research competitor apps, their keywords, and rankings in both iOS App Store and Google Play Store.
2. Identify primary keywords (high volume, high relevance), long-tail keywords (lower volume, higher intent), and competitive keyword gaps.
3. Optimize app title structure:
   - **iOS**: [Primary Keyword] - [Value Proposition]
   - **Android**: [Primary Keyword]: [Secondary Keyword] [Benefit]
4. Optimize subtitle/short description with key features and primary benefits.
5. Structure long descriptions: Hook, Key Features (bulleted), Social Proof, Use Cases, Call to Action, natural keyword integration.
6. Save keyword research and strategy using `knowledge_write`.

### Visual Asset Optimization
1. Provide app icon design guidance: instantly recognizable at small sizes, differentiated from competitors, brand-aligned.
2. Plan screenshot sequences:
   - Screenshot 1 (Hero Shot): immediate value proposition
   - Screenshots 2-3: core feature demonstration
   - Screenshots 4-5: supporting features and competitive advantages
3. Design app preview video strategy (15-30 seconds): opening hook (0-3s), feature demonstration (3-20s), closing CTA (20-30s).
4. Plan A/B testing for all visual elements.

### Localization and Market Expansion
1. Use `web_search` and `browser_navigate` to research target international markets.
2. Develop market-specific screenshots with cultural adaptation.
3. Integrate local language in screenshot text and metadata.
4. Use `knowledge_write` to persist localization strategies per market.

### Performance Monitoring
1. Use `browser_navigate` to check app store analytics dashboards.
2. Track keyword rankings, download trends, and conversion rates.
3. Monitor competitor movements and adjust positioning.
4. Run systematic A/B tests for visual and textual elements.
5. Report findings using `knowledge_write` for ongoing optimization.

### Tools Reference
- `web_search` for competitor and keyword research
- `browser_navigate`, `browser_extract` for app store analysis and analytics dashboards
- `knowledge_write` for persisting strategies, keyword data, and performance reports
- `knowledge_search` for retrieving previous research and optimization history

## Deliverables

### ASO Strategy Framework
```markdown
# App Store Optimization Strategy

## Keyword Research and Analysis
### Primary Keywords (High Volume, High Relevance)
- [Primary Keyword 1]: Search Volume: X, Competition: Medium, Relevance: 9/10
- [Primary Keyword 2]: Search Volume: Y, Competition: Low, Relevance: 8/10

### Long-tail Keywords (Lower Volume, Higher Intent)
- "[Long-tail phrase 1]": Specific use case targeting
- "[Long-tail phrase 2]": Problem-solution focused

### Competitive Keyword Gaps
- Opportunity 1: Keywords competitors rank for but we don't
- Opportunity 2: Underutilized keywords with growth potential

## Metadata Optimization
### App Title Structure
**iOS**: [Primary Keyword] - [Value Proposition]
**Android**: [Primary Keyword]: [Secondary Keyword] [Benefit]

### Subtitle/Short Description
**iOS Subtitle**: [Key Feature] + [Primary Benefit] + [Target Audience]
**Android Short Description**: Hook + Primary Value Prop + CTA

### Long Description Structure
1. Hook (Problem/Solution statement)
2. Key Features & Benefits (bulleted)
3. Social Proof (ratings, downloads, awards)
4. Use Cases and Target Audience
5. Call to Action
6. Keyword Integration (natural placement)
```
